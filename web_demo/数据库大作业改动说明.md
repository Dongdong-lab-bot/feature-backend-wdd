# 数据库大作业改动说明

## 一、新增项目概述

新增 `web_demo/` 目录，构建了一个**食品安全管理系统数据库设计 Web Demo**，包含：
- 后端 CRUD API（FastAPI + SQLite）
- 前端单页应用（纯 HTML/JS/CSS）
- SQL 查询面板（直接写 SQL 看数据库）
- 完整测试脚本

---

## 二、数据库设计（6 张表）

```
stores ──1:N── cameras ──1:N── alerts
                                  │1:1
                                  ├── violation_events
                                  │N:1
                                  └── rectification_tasks
users ──独立用户表（含 role_type / store_ids 权限设计）
```

**核心知识点**：实体完整性（PK）、参照完整性（FK）、逻辑删除、数据类型、默认值

| 预设 | SQL 演示内容 |

| 3 表联合查询（JOIN） | `alerts LEFT JOIN stores LEFT JOIN cameras` — 参照完整性 |

| 外键关联检查 | `LEFT JOIN + CASE WHEN` — 参照完整性验证 |
| 待处理工单详情 | 4 表 JOIN — 综合查询 |

## 六、启动方式

启动服务后访问 http://localhost:8080/demo

**使用方法**：
1. 登录页输入演示账号（admin01/super01/mgr001，密码123456）
2. 以不同角色查看告警/事件/工单，体验权限差异
3. 点击左侧 **SQL 查询** 进入 SQL 面板
4. 点击预设按钮（如"3 表联合查询"）一键看结果
5. 或自己在编辑框写 SQL，点击"执行 SQL"
6. 切回其他页面查看业务数据增删改查
