# 数据库大作业改动说明

## 简化方案

从原项目（食品安全管理系统）简化，聚焦大作业要求的基本功能。

### 数据库表（6张 → 4张）

| 原表 | 简化后 | 说明 |
|-----|-------|------|
| stores | ✅ stores（门店表） | 保留 |
| cameras | ❌ 删除 | 合并到违规记录 |
| alerts | ✅ violations（违规记录表） | 改名，核心表 |
| violation_events | ❌ 删除 | 合并到违规记录 |
| users | ✅ users（用户表） | 保留 |
| rectification_tasks | ✅ tasks（整改工单表） | 改名 |

### 功能页面（7个 → 6个）

1. **Dashboard** — 统计概览（违规总数、严重违规、待处理工单、已完成工单）
2. **违规记录** — 增删改查（核心 CRUD，含分页、筛选、逻辑删除）
3. **整改工单** — 状态管理（待处理/处理中/已完成）
4. **门店管理** — 门店列表展示
5. **SQL 查询** — 自定义 SQL + 预设演示（JOIN、GROUP BY、逻辑删除、外键检查）
6. **E-R 关系图** — 可视化展示 4 张表结构及外键关系

### 保留的数据库知识点

- ✅ 实体完整性（PRIMARY KEY, UNIQUE）
- ✅ 参照完整性（FOREIGN KEY）
- ✅ 逻辑删除（is_deleted）
- ✅ 多表联合查询（JOIN）
- ✅ 分组统计（GROUP BY）
- ✅ 数据库安全（仅允许 SELECT）
- ✅ 角色权限（3 种角色数据隔离）

### 启动方式

```bash
python serve_demo.py
# 访问 http://localhost:8000
```
