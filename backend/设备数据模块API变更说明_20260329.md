# 设备数据模块 API 变更说明（2026-03-29）

本文档基于当前后端实现，说明本次交付的 3 个接口以及 1 个已删除接口。

## 统一返回约定

普通 JSON 接口：

```json
{
  "code": 200,
  "msg": "success",
  "request_id": null,
  "data": {}
}
```

业务异常：HTTP 状态码为 4xx/5xx，`detail` 内包含：

```json
{
  "code": 40000,
  "msg": "错误信息",
  "data": null
}
```

## 1. 获取设备树

- 接口：`GET /devices/tree`
- 功能：返回当前用户可见范围内的组织树，并在食堂节点挂载设备列表。
- 权限范围：
  - 监管端：可见当前租户下全部食堂设备（并返回对应组织层级）。
  - 执行端：按当前用户组织向下收敛到食堂范围。

### 请求参数

无。

### 响应体

`data` 为数组（树根节点列表）：

| 字段 | 类型 | 说明 |
|---|---|---|
| id | int | 组织 ID |
| name | string | 组织名称 |
| org_type | string | 组织类型：AREA/SCHOOL/CANTEEN |
| parent_id | int/null | 父组织 ID |
| children | array | 子组织节点 |
| devices | array | 当前组织下设备（仅食堂节点一般有值） |

`devices[]` 字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| id | int | 设备 ID |
| device_code | string | 设备编码 |
| device_name | string | 设备名称 |
| status | string | ONLINE/OFFLINE/MAINTENANCE |
| last_heartbeat | string/null | 最近心跳时间（ISO 时间） |

### 响应示例

```json
{
  "code": 200,
  "msg": "success",
  "data": [
    {
      "id": 1,
      "name": "朝阳区",
      "org_type": "AREA",
      "parent_id": null,
      "children": [
        {
          "id": 2,
          "name": "北京市第一中学",
          "org_type": "SCHOOL",
          "parent_id": 1,
          "children": [
            {
              "id": 3,
              "name": "第一食堂",
              "org_type": "CANTEEN",
              "parent_id": 2,
              "children": [],
              "devices": [
                {
                  "id": 101,
                  "device_code": "AC1256EF456E",
                  "device_name": "手部晨检仪",
                  "status": "ONLINE",
                  "last_heartbeat": "2026-03-29T08:30:00Z"
                }
              ]
            }
          ],
          "devices": []
        }
      ],
      "devices": []
    }
  ]
}
```

## 2. 编辑设备消息记录

- 接口：`PUT /devices/records/{id}`
- 功能：编辑设备消息记录的业务字段（状态、关联台账标识、payload 等）。

### 路径参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | int | 是 | 设备消息记录 ID |

### 请求体（全部可选，至少传 1 个字段）

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| is_related_ledger | bool | 否 | 是否关联电子台账 |
| status | string | 否 | 设备状态：ONLINE/OFFLINE/MAINTENANCE |
| data_type | string | 否 | 数据类型：MORNING_CHECK_RECORD/AI_BOX_RECORD |
| source | string | 否 | 来源标识，最大 32 字符 |
| process_result | string | 否 | 处理结果，最大 32 字符 |
| submit_date | string(date) | 否 | 提交日期，格式 YYYY-MM-DD |
| payload | object | 否 | 业务摘要数据 |
| detail_json | object | 否 | 明细原始数据 |

### 请求示例

```json
{
  "is_related_ledger": true,
  "status": "ONLINE",
  "process_result": "SUCCESS",
  "payload": {
    "temperature": 36.6,
    "has_mask": true
  }
}
```

### 响应体

| 字段 | 类型 | 说明 |
|---|---|---|
| id | int | 记录 ID |
| is_related_ledger | bool | 是否关联电子台账 |
| status | string | 当前记录状态 |
| updated_at | string | 更新时间（ISO 时间） |

### 响应示例

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "id": 1001,
    "is_related_ledger": true,
    "status": "ONLINE",
    "updated_at": "2026-03-29T09:10:00Z"
  }
}
```

### 典型错误

- `40000`：请求参数非法，或请求体为空。
- `40402`：记录不存在或无可见权限。
- `50008`：编辑失败。

## 3. 导出设备消息记录

- 接口：`GET /devices/records/export`
- 功能：按日期范围导出设备消息记录。
- 返回：文件流下载（非 JSON）。

### Query 参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| start_date | string(date) | 否 | 开始日期，YYYY-MM-DD |
| end_date | string(date) | 否 | 结束日期，YYYY-MM-DD |
| format | string | 否 | 导出格式：excel/pdf/csv，默认 excel |

### 导出字段

1. 记录ID
2. 提交食堂
3. 设备名称
4. 设备编码
5. 设备数据类型
6. 是否关联电子台账
7. 提交日期
8. 设备状态
9. 来源
10. 处理结果
11. 追踪ID

### 响应头示例

- `Content-Type`：
  - excel: `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`
  - pdf: `application/pdf`
  - csv: `text/csv`
- `Content-Disposition`：`attachment; filename="device_records_YYYYMMDD.xxx"`

### 典型错误

- `40000`：日期非法、`start_date > end_date`、format 不在允许范围。

## 4. 已删除接口（重点标记）

### `GET /devices`（设备列表接口）

- 状态：**已删除（本次变更移除）**
- 原能力：设备分页列表查询（支持状态、组织、关键字筛选）。
- 影响：前端/调用方需改为通过以下接口获取数据：
  - 设备树：`GET /devices/tree`
  - 单设备详情：`GET /devices/{id}`

> 说明：删除后，调用 `GET /devices` 将不再命中设备模块路由。
