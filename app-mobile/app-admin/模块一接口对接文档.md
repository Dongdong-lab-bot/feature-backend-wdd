## 对接说明

- 认证要求：除登录/公开接口外，需携带 Authorization: Bearer <token> 与 X-App-Client 头，见 auth.py 。
- 通用响应格式： {"code": 200, "msg": "success", "data": ...} ，见多处返回结构 api.py 。
- 角色与租户隔离：所有 admin 接口均依赖 get_current_user ，并在查询/写入中使用 tenant_id 过滤。

## 错误码清单（常见）

- 200：成功（统一 ok 响应）
- 400：请求参数或业务校验失败（例如用户名重复、无效字段）
- 401：未认证或 token 无效
- 403：无权限或鉴权失败
- 404：资源不存在
- 409：冲突（例如手机号已注册）
- 429：验证码发送频率限制
- 40301：跨租户访问被拒绝（TenantAccessError）
- 40302：安全拦截器拒绝（SecurityException）
- 40002：数据完整性冲突（唯一约束/外键约束等）
- 46001：验证码错误
- 46002：验证码过期
- 46003：验证码验证次数超限
- 500：服务器内部错误
   参考： exceptions.py
## 字段枚举值说明

- Org 部门类型（org_type）： AREA | SCHOOL | CANTEEN
  - 见 schemas.py:L138-L148
- 用户状态（status）： ACTIVE | DISABLED
  - 见 schemas.py:L45-L47
- 角色类型（role_type）： REGULATOR | EXECUTOR
  - 见 schemas.py:L83-L89
- 菜单隐藏（hidden）： true | false
  - 见 schemas.py:L161-L177

### 修改密码 (PUT /auth/password)

- **鉴权**：需要登录（携带 `Authorization: Bearer <accessToken>`）
- **场景**：已登录用户修改自己的密码，需验证旧密码
- **请求头**：建议携带 `X-App-Client: reg_app`
- **请求体 (JSON)**:
  ```json
  {
    "oldPassword": "OldPassword123",
    "newPassword": "NewPassword456"
  }
  ```
- **错误响应**:
  - 旧密码错误：
    ```json
    {
      "code": 400,
      "msg": "incorrect old password",
      "data": null
    }
    ```
  - 新密码无效：
    ```json
    {
      "code": 400,
      "msg": "invalid password",
      "data": null
    }
    ```
  - 新旧密码相同：
    ```json
    {
      "code": 400,
      "msg": "new password cannot be the same as old password",
      "data": null
    }
    ```
- **响应体 (JSON)**:
  ```json
  {
    "code": 200,
    "msg": "success",
    "data": {
      "updated": true
    }
  }
  ```

### 获取当前用户信息 (GET /auth/me)

- **鉴权**：需要登录（携带 `Authorization: Bearer <accessToken>`）
- **请求头**：建议携带 `X-App-Client: reg_app`
- **说明**：返回当前登录用户的完整信息，包含基本资料、所属辖区、部门及职务信息
- **响应体 (JSON)**:
  ```json
  {
    "code": 200,
    "msg": "success",
    "data": {
      "id": 5,
      "username": "13800000001",
      "realName": "张三",
      "mobile": "13800000001",
      "gender": "男",
      "birthday": "1990-01-01",
      "email": "zhangsan@example.com",
      "canteenScope": "北京市朝阳区",
      "orgId": 3,
      "orgName": "朝阳区市场监管局",
      "roleId": 2,
      "roleName": "局长",
      "roleType": "REGULATOR",
      "tenantId": 1,
      "status": "ACTIVE"
    }
  }
  ```
- **字段说明**：
  - `id`: 用户ID
  - `username`: 用户名
  - `realName`: 真实姓名
  - `mobile`: 手机号码
  - `gender`: 性别
  - `birthday`: 生日
  - `email`: 邮箱地址
  - `canteenScope`: 所属辖区
  - `orgId`: 部门ID
  - `orgName`: 部门名称（关联查询）
  - `roleId`: 职务ID
  - `roleName`: 职务名称（关联查询 Role 表的 name 字段）
  - `roleType`: 角色类型（REGULATOR 监管方 | EXECUTOR 执行方）
  - `tenantId`: 租户ID
  - `status`: 账号状态（ACTIVE 正常 | DISABLED 禁用）
