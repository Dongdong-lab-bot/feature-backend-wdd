## 4. HBuilderX 真机/模拟器：网络配置与测试（重点）

真机/模拟器下最常见的问题：**连不上后端**。根因：手机里的 `127.0.0.1` 是手机自己，不是你的电脑。

App 请求会读取：

- `uni.getStorageSync('baseUrl')`
  若为空，会默认 `http://127.0.0.1:8000`（真机会失败）

因此真机/模拟器必须把 baseUrl 设置为你电脑的"局域网 IP"。

### 4.1 第一步：保证手机和电脑在同一个 Wi‑Fi

- 手机连 Wi‑Fi A
- 电脑也连 Wi‑Fi A
- 不建议：电脑网线、手机 Wi‑Fi（容易跨网段/隔离）

### 4.2 第二步：获取电脑的局域网 IP（macOS）

在终端执行：

```bash
ipconfig getifaddr en0
```
假设输出为 `192.168.1.10`，记为 `LAN_IP`。

### 4.3 第三步：确认后端监听所有网卡（必须）

后端启动必须带：

- `--host 0.0.0.0`

示例：

```bash
cd /Users/ruaww/KP/Safefood-Platform/backend
source .venv/bin/activate
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 4.4 第四步：用手机浏览器验证"能访问到后端"

在手机浏览器打开：

- http://LAN_IP:8000/docs

如果打不开，不要继续跑 App，先排查：

- macOS 防火墙是否拦截 Python/uvicorn
- 公司 Wi‑Fi 是否"客户端隔离"（手机不能访问局域网设备）
- 手机/电脑是否确实同一网段

### 4.5 第五步：安装前端依赖

```bash
cd 你的项目文档地址/Safefood-Platform/app-mobile/app-admin
npm install --workspaces=false --legacy-peer-deps

cd 你的项目文档地址/Safefood-Platform/app-mobile/app-execution
npm install --workspaces=false --legacy-peer-deps
```

### 4.6 第六步：用 HBuilderX 运行到真机/模拟器

1. 打开 HBuilderX
2. 打开工程目录（两端分别开）：
   - 监管端：/Users/ruaww/KP/Safefood-Platform/app-mobile/app-admin
   - 执行端：/Users/ruaww/KP/Safefood-Platform/app-mobile/app-execution
3. 菜单选择：**运行 -> 运行到手机或模拟器**
4. 等待安装/启动完成

如果你此时在登录时提示“网络错误”，优先按 4.6 配置 baseUrl（真机不能用 127.0.0.1）。

### 4.7 第七步：设置 baseUrl（最关键）

真机/模拟器不能用 `127.0.0.1`，必须把 baseUrl 设置为电脑局域网地址：
- `http://LAN_IP:8000`

#### 方式 A（推荐）：在 App 内设置（无需 HBuilderX 控制台）

如果你还没登录进不去“我的”，直接在登录页操作：
- 监管端 App：登录页点击 **接口地址设置** → 修改 baseUrl → **保存** → **测试连接**
- 执行端 App：登录页点击 **接口地址设置** → 修改 baseUrl → **保存** → **测试连接**

也可以登录后在“我的”里修改：
- 监管端 App：进入 **我的** → **设置** → **环境配置** → 修改 baseUrl → **保存**
- 执行端 App：进入 **我的** → **设置** → 修改 baseUrl → **保存**

测试连接会请求 `${baseUrl}/health`，返回 200 即表示网络 OK。

#### 方式 B（可选）：用 HBuilderX 控制台设置

如果你能找到 HBuilderX 的控制台/调试控制台，也可以执行：

```js
uni.setStorageSync('baseUrl', 'http://LAN_IP:8000')
uni.getStorageSync('baseUrl')
```

建议操作：
- 第一次写入后，**手动杀掉 App 再打开**（或在 HBuilderX 再运行一次），确保所有页面用到的新 baseUrl 生效。

### 4.8 第八步：确认 appClient 正确（避免 403）

后端要求请求头：

- `X-App-Client: reg_app` 或 `exec_app`

App 启动时会写入 appClient，但你仍可在控制台确认：

```js
uni.getStorageSync('appClient')
```

如果不对，强制写入：

- 监管端 App：
```js
uni.setStorageSync('appClient', 'reg_app')
```

- 执行端 App：
```js
uni.setStorageSync('appClient', 'exec_app')
```

## 5. 真机常见问题排查（按出现频率排序）

### 5.1 手机浏览器打不开 http://LAN_IP:8000/docs

- 后端没有 `--host 0.0.0.0`
- macOS 防火墙拦截
- 公司 Wi‑Fi 客户端隔离（换手机热点/家用路由器测试）

### 5.2 手机能打开，但 App 仍提示网络错误

- baseUrl 没写入成功：执行 `uni.getStorageSync('baseUrl')` 验证
- baseUrl 少了 `http://`：必须是 `http://LAN_IP:8000`
- App 没重启：杀掉重开或重新运行

### 5.3 App 登录 403

- 通常是 appClient 不匹配：确认
  - 监管端 App：reg_app
  - 执行端 App：exec_app

### 5.4 App 登录 401

- 账号密码错误：确认 admin/py427123
- 后端没正常启动：先 curl /health
- 后端启动报错：优先修复后端启动日志

## 7. 常见问题补充

### 7.1 npm install 失败

- 遇到 peer dependency 冲突：添加 `--legacy-peer-deps` 参数
- 遇到 404 错误：检查 package.json 是否有无效依赖（如 @wot-design-uni/components）

### 7.2 端口冲突

- Vite 端口被占用时会自动递增（3000 → 3001 → 3002）
- 具体端口号请查看控制台输出

### 7.3 数据库初始化

- 后端首次启动会自动创建 SQLite 数据库（dev.db）
- 如需重置：删除 backend/dev.db 后重启后端
