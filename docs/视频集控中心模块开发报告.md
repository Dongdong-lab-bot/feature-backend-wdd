## 一、开发概述

### 1.1 需求背景
根据项目需求，需要实现视频集控中心模块，提供食堂/区域下摄像头树获取、实时流播放地址获取、录像回看地址获取、抓拍并落库、视频巡检结果提交与查询等核心功能。

### 1.2 开发范围
- 后端：FastAPI 异步接口开发
- 数据库：SQLAlchemy ORM 模型设计

## 四、核心业务逻辑

### 4.1 摄像头树获取
1. 根据 tenant_id 查询所有启用的摄像头
2. 按照 canteen_id 构建树形结构：片区 → 食堂 → 摄像头
3. 返回嵌套的树形结构

### 4.2 实时流地址获取
1. 根据 camera_id 查询摄像头信息
2. 生成 Stream Key：`stream_{camera_id}`
3. 返回 HLS、RTMP、FLV 等多协议播放地址

### 4.3 回放地址获取
1. 根据 camera_id、begin、end 时间段生成回放 Stream Key
2. 格式：`playback_{camera_id}_{begin_timestamp}`
3. 返回 HLS 播放地址及时间戳信息

### 4.4 抓拍功能
1. 接收前端 Base64 图片数据
2. 解码并保存到本地 uploads/captures 目录
3. 记录抓拍元数据到 biz_video_capture 表
4. 返回照片URL

### 4.5 巡检结果提交流程
1. 根据 template_id 查询模板获取大项配置
2. 计算总分、红线/黄线问题数
3. 保存表单快照和检查结果
4. 创建审核日志记录
5. 返回任务ID和状态

## 七、后续优化建议

1. **流媒体服务器对接**: 当前播放参数为模拟数据，需接入真实流媒体服务器(如ZLMediaKit、SRS等)
2. **OSS存储集成**: 抓拍图片建议统一存储到OSS而非本地文件系统
3. **海康威视SDK对接**: 接入真实海康云眸SDK获取设备通道信息
4. **定时任务**: 视频巡检任务的自动生成与派发
5. **消息通知**: 巡检结果提交时的消息推送
6. **数据导出**: 支持导出Excel/PDF格式的巡检报告
