## 后续增补（契约对齐与验证记录）

- 统一上下文/JWT 类型：tenant/user 改为 int，system_mode 改为布尔默认 False。
- 增加 ORM 租户拦截器：`do_orm_execute` 自动注入 tenant 过滤，无上下文拒绝查询。
- Auth 中间件：仅 `role_type=system` 时进入上帝模式，解析 ID 为整型。
- 中间件顺序：Auth 最外层，其次事件，再 CORS。
- 组织树查询：ORM CTE + aliased，显式带 tenant，避免 `from_statement` 绕过拦截。
- 服务接口签名：tenant/user 参数改为 int，JWT 测试签发同步改为字符串编码。
- 验证：系统 Python 环境下开启 system_mode 并设置 tenant 后，成功创建根组织并查询树，生成可用 Bearer token。
