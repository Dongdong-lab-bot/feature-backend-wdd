# 食安大模型Agent集群项目 周度细化研发计划&里程碑文档

**文档版本**：V1.1
**适用周期**：6周MVP全闭环交付
**核心对齐原则**：

1. 严格遵循4个组封顶的管理要求，不新增分组、不扩大管理半径
2. 任务拆解到单周，每周目标闭环、里程碑可量化验收，无模糊需求
3. 全程贴合新人带教需求，每周明确新人专属上手任务，低门槛高参与度
4. 全周期死守4条MVP红线，每周设置合规校验节点，从根源规避架构腐化

---

## 前置全局对齐

### 固定分组与职责边界（全程不调整）

| 组别                 | 组长角色        | 核心唯一职责                                                         | 专属代码阵地                                                                      |
| :------------------- | :-------------- | :------------------------------------------------------------------- | :-------------------------------------------------------------------------------- |
| 🏗️ 核心基建组      | 架构师/后端主程 | 全局标准制定、底层基建搭建、公共能力封装、边缘数据接入、红线规则落地 | `src/core/`、`src/api/`、`src/edge/`、`infra/`                            |
| 🛡️ 监管研判Agent组 | 算法/后端负责人 | YOLO告警降噪、VLM二次校验、违规分级、整改核验核心逻辑开发            | `src/agents/triage_agent/`、`src/prompts/triage/`                             |
| 📊 数据分析Agent组   | 后端/数据负责人 | Text-to-SQL核心引擎开发、数据安全管控、合规报表生成                  | `src/agents/data_agent/`、`src/tools/db_reader.py`、`src/prompts/data/`     |
| 🧠 中枢路由Agent组   | 产品/后端负责人 | 用户交互入口、意图识别、多Agent路由分发、多角色差异化交互            | `src/agents/router_agent/`、`src/api/chat_router.py`、`src/prompts/router/` |

### 全程死守4条红线（每周校验，违反一票否决周度里程碑）

1. **Schema统一红线**：所有数据流转必须严格遵循 `src/core/schemas.py`定义的格式，任何组不得私自修改
2. **数据安全红线**：数据分析Agent仅能使用只读数据库账号，无任何写库权限，所有写操作仅能由基建组硬编码API实现
3. **稳定性红线**：所有大模型调用必须配置超时重试、熔断降级逻辑，连续3次调用失败必须切硬编码兜底
4. **可追溯红线**：所有大模型判定、数据查询、工单操作必须写入带时间戳的审计日志，全链路可追溯
