# 模块二：数字化台账 & 视频巡检 - 架构强制契约

### 🚫 零容忍红线 (Zero Tolerance)

1.  **严禁跨模块查表**: 模块二的代码**绝对不允许**写 SQL 去 join 模块一的 `sys_org` 或 `sys_user` 表。必须调用 Service 接口。
2.  **严禁硬编码**: IP 地址、端口、密码、密钥必须配置在 `config.yaml` 或环境变量中。
3.  **严禁私自修改状态**: 台账状态变更必须经过状态机校验, **禁止**手动 `UPDATE status = 'xxx'`。

### 1. 💾 数据存储契约 (Storage Spec)
**对接人**: 麦锦涛 & 张子昊 & 谢传宇

#### 1.1 JSONB 扁平化规范
为了支持 PostgreSQL 的 GIN 索引, `ledger_instance.content` 必须使用**扁平化 Key-Value** 结构。

*   **❌ 错误写法 (嵌套过深,无法索引):**
    ```json
    {
      "morning_check": {
        "temperature": "36.5",
        "staff": { "name": "张三" }
      }
    }
    ```

*   **✅ 正确写法 (Key = Schema 中的 field_id):**
    ```json
    {
      "f_temp_01": "36.5",
      "f_staff_name": "张三",
      "f_is_clean": true
    }
    ```

#### 1.2 快照冗余规范
读取历史台账时, **严禁查询 `ledger_template` 表**。

*   **规定**: `ledger_instance` 表必须包含 `schema_snapshot` 字段。
*   **逻辑**: 任务生成时刻, 将当时的模板完整 Copy 进快照。渲染历史数据**只读快照**。

### 2. 🤖 状态机契约 (State Machine Spec)
**对接人**: 李文钊 & 谢传宇

必须在 `modules/ledger/constants.py` 中定义, 全员引用。

```python
from enum import Enum

class LedgerStatus(str, Enum):
    PENDING = "PENDING"     # 待填报(系统生成)
    FILLING = "FILLING"     # 填报中(暂存)
    SIGNED = "SIGNED"       # 已签字(提交,不可改)
    ARCHIVED = "ARCHIVED"   # 已归档 (历史数据)

# 允许的流转路径
ALLOWED_TRANSITIONS = {
    LedgerStatus.PENDING: [LedgerStatus.FILLING, LedgerStatus.SIGNED],
    LedgerStatus.FILLING: [LedgerStatus.SIGNED],
    LedgerStatus.SIGNED: [LedgerStatus.ARCHIVED],
    LedgerStatus.ARCHIVED: [] # 终态
}
```

### 3. 🛡️ 安全与存证契约 (Security Spec)
**对接人**: 谢传宇 & 李文钊

#### 3.1 硬件数据隔离
*   **规定**: 硬件设备 (蓝牙秤、晨检仪) **严禁直接 UPDATE `ledger_instance` 表**。
*   **路径**: 硬件 -> `device_buffer` 表 -> (用户打开页面) -> 读取 Buffer 自动填充 -> 用户确认提交。

#### 3.2 签字防篡改 (Hash)
在状态变为 `SIGNED` 的瞬间, 必须计算并锁定 Hash。

```python
import hashlib
import json

def calculate_security_hash(content: dict, snapshot: dict, salt: str) -> str:
    # 这里的 json.dumps 必须指定 sort_keys=True 以保证顺序一致
    raw = json.dumps(content, sort_keys=True) + json.dumps(snapshot, sort_keys=True) + salt
    return hashlib.sha256(raw.encode()).hexdigest()
```

### 4. 📺 视频流媒体契约 (Streaming Spec)
**对接人**: 万鹏皓 & 黎运哲

#### 4.1 Stream Key 命名规则
流媒体服务器与业务后台必须遵守同一套命名规则, 否则前端无法播放。

*   **格式**: `t_{tenant_id}_c_{camera_id}`
*   **示例**: 租户1001的5号摄像头 -> `t_1001_c_5`

#### 4.2 播放地址生成
*   **HLS 地址**: `http://{media_server_ip}:8888/{stream_key}/index.m3u8`
*   **WebRTC 地址**: `http://{media_server_ip}:8889/{stream_key}`

#### 4.3 延迟红线
*   **规定**: 端到端延迟必须 **< 1000ms**。
*   **措施**: MediaMTX 配置文件中必须开启 `hlsAlwaysRemux: no` 和 `lowLatency: yes`。
