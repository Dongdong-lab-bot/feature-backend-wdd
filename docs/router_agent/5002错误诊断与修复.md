# 5002 错误诊断与修复

## 现象

通过 curl 调用 `POST /api/v1/chat/messages` 时，所有正常路由返回 `5002` 错误。

```json
{"code": "5002", "message": "路由处理失败", "success": false, "response_type": "error", ...}
```

## 定位

1. 诊断脚本 `scripts/diagnose_5002.py` 和 `scripts/test_api_chain.py` 逐级排查
2. 发现 `handle_chat_router()` 内部触发了 `AttributeError`
3. 具体错误：
   ```
   'RouterSessionManager' object has no attribute '_redis_usable'
   ```

## 根因

`src/agents/router_agent/context_manager.py` 里的 `RouterSessionManager` 调用了两个**不存在**的方法：

| 调用的方法 | 实际缺失 |
|------------|----------|
| `self._redis_usable()` | 未定义 |
| `self._mark_redis_unavailable(exc)` | 未定义 |

**触发条件**：本地开发环境没有 Redis，`_ensure_redis()` 连接到 Redis 失败后，`cleanup_expired_confirmations()` 等代码调用了这两个方法，引发了 `AttributeError`。

**为什么返回 5002**：`chat_router.py` 的 `try/except Exception` 顶兜底捕获了该 `AttributeError`，统一转换成 `code="5002"` 的错误响应，掩盖了真正的原因。

## 修复

在 `RouterSessionManager` 中补上两个缺失方法：

```python
def _redis_usable(self) -> bool:
    """Redis 是否可用的简便检查。"""
    return not self._redis_unavailable and self._redis is not None

def _mark_redis_unavailable(self, exc: Exception) -> None:
    """标记 Redis 不可用，并记录日志。"""
    self._redis_unavailable = True
    log.warning(f"Redis operation failed, marked unavailable: {exc}")
```

## 验证

修复后全部通过测试：

| 测试 | 预期 | 结果 |
|------|------|------|
| 创建会话 | `0000` | ✅ |
| 违规查询 (关键词兜底) | `query_detail` | ✅ |
| OOD 天气 | `out_of_domain` | ✅ |
| 违规排名 | `query_ranking` | ✅ |
| 创建整改工单 | `create_task` | ✅ |

## 结论

- 5002 不是逻辑错误，是**方法缺失**导致的对象属性崩溃
- 修复后无论 Redis 是否可用，关键词兜底和路由链路都能正常工作
- LLM 路由因 `deepseek-reasoner` 不支持 `tool_choice` 而降级到关键词，这是另一回事，不影响主链路
