### 四、 组长每日“整合”工作手册

**为了避免混乱，建议两位组长每日严格执行以下流程：**

* **代码提交规范：** **所有的组员不允许直接推送到** feature/frontedn/main**分支。必须提交到自己的分支（如** feature/frontedn/xxx），然后由组长在本地 **git merge** **检查后再推送。**
* **公共组件抽离：**

  * **例如：监管端和执行端都有“职务管理”。**
  * **操作：** **两位组长在周二下班前沟通一下，如果 UI 长得一样，由一人写好组件放到** **common**，另一人直接调用，避免重复劳动。
