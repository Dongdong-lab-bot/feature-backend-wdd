## 一、表清单

| 表名 | 对应 Pydantic 模型 | 职责 | 物理主键 | 业务唯一键 |
|------|-------------------|------|---------|-----------|
| `alerts` | `AlertCoreModel` | 边缘告警原始数据 | `id` VARCHAR(32) | `id` |
| `violation_events` | `ViolationEventModel` | VLM 二次校验结论 | `id` BIGINT AUTO_INCREMENT | `event_id` VARCHAR(32) |
| `users` | `UserInfoModel` | 用户身份与权限 | `id` BIGINT AUTO_INCREMENT | `user_id` VARCHAR(32) |
| `audit_logs` | `AuditLogModel` | 全链路审计 | `id` BIGINT AUTO_INCREMENT | `audit_id` VARCHAR(32) |
| `confirmations` | `ConfirmationModel` | 高危操作二次确认 | `id` BIGINT AUTO_INCREMENT | `confirmation_id` VARCHAR(32) |

**暂不落地（Redis 缓存）**：`SessionContext`、`RoutePlanModel`

## 四、关联关系

```
alerts (1) ←——— (N) violation_events
    │                    │
    │   alert_id (FK)    │
    │   ON DELETE        │
    │   RESTRICT         │
    │                    │
    └── store_id 对齐 ───┘
```

**仅 `violation_events.alert_id → alerts.id` 存在外键约束**，其余表保持松耦合，便于独立维护和并行开发。

**所有业务查询默认过滤条件**：`WHERE is_deleted = 0`

## 五、数据 Agent 常用查询示例

### 5.1 按门店+时间范围查询告警列表（带权限过滤）

```sql
-- 数据 Agent SQL 生成典型场景：用户问"南山店最近7天有哪些告警"
-- permission_context.store_scope = ['STORE_001']
SELECT
    id, camera_id, camera_name, location, message,
    violation_type, level, confidence, image_url,
    rectification_status, timestamp
FROM alerts
WHERE is_deleted = 0
  AND store_id IN ('STORE_001')           -- 权限过滤
  AND timestamp >= '2026-04-23 00:00:00'  -- 时间范围
  AND timestamp < '2026-04-30 00:00:00'
ORDER BY timestamp DESC
LIMIT 20;
```

### 5.2 按违规状态查询事件（监管端核心查询）

```sql
-- 监管端：查询"南山店已确认的违规事件"
SELECT
    e.event_id, e.alert_id, e.store_id, e.message,
    e.severity_level, e.verification_method, e.vlm_confidence,
    e.model_version, e.verified_at, e.determination_basis
FROM violation_events e
WHERE is_deleted = 0
  AND is_violation_confirmed = 1            -- 仅确认违规
  AND store_id IN ('STORE_001')             -- 权限过滤
  AND timestamp >= '2026-04-01 00:00:00'
ORDER BY timestamp DESC
LIMIT 50;
```

### 5.3 告警+事件联合查询（详情页）

```sql
-- 前端详情页：查询某条告警及其 VLM 校验结论
SELECT
    a.id AS alert_id, a.camera_id, a.message, a.level AS alert_level,
    a.confidence AS yolo_confidence, a.image_url, a.bbox,
    a.rectification_status, a.timestamp AS alert_time,
    e.event_id, e.is_violation_confirmed, e.severity_level,
    e.vlm_confidence, e.model_version, e.verified_at,
    e.determination_basis, e.regulation_clause
FROM alerts a
LEFT JOIN violation_events e
    ON a.id = e.alert_id AND e.is_deleted = 0
WHERE a.is_deleted = 0
  AND a.id = 'ALT1745733600123A7F9X2K1';
```

### 5.4 用户权限查询（登录时加载）

```sql
-- 用户登录时加载权限上下文
SELECT
    user_id, user_name, role_type,
    store_ids, region_ids, permission_actions
FROM users
WHERE is_deleted = 0
  AND user_id = 'USER001';
```

### 5.5 审计日志按 trace_id 全链路追溯

```sql
-- 问题排查：按 trace_id 串联全链路操作
SELECT
    event_type, operator, action, success,
    latency_ms, error_code, event_time
FROM audit_logs
WHERE is_deleted = 0
  AND trace_id = 'a1b2c3d4e5f6789012345678abcdef03'
ORDER BY event_time ASC;
```

### 5.6 用户待处理确认单查询

```sql
-- 前端轮询：查询用户待确认的过期操作
SELECT
    confirmation_id, action_type, action_preview,
    action_params, confirmation_status, expires_at
FROM confirmations
WHERE is_deleted = 0
  AND user_id = 'USER002'
  AND confirmation_status = 'pending'
  AND expires_at > NOW()
ORDER BY created_at DESC;
```

## 七、红线提醒

1. **禁止物理删除**：所有删除操作必须更新 `is_deleted = 1`，严禁 `DELETE FROM` 任何核心表
2. **禁止裸字符串枚举**：所有枚举字段（`level`、`violation_type`、`role_type` 等）必须使用 Pydantic Enum 常量，禁止代码中自由拼写字符串
3. **时间带时区**：所有 `DATETIME(3)` 字段存储北京时间（UTC+8），`mysql_init.sh` 已强制设置会话时区
4. **敏感信息脱敏**：`audit_logs.input_data/output_data` 落库前必须经过统一脱敏工具处理
5. **权限过滤字段**：数据 Agent SQL 中的权限过滤只能使用 `store_id`，禁止使用 `location` 等展示字段
