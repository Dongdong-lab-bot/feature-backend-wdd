## 二、alert_data 原始 JSON 结构

EdgeAgent 构造的原始 payload：

```json
{
  "camera_id": "camera_001",
  "message": "[no_mask] 检测到: no_mask",
  "level": "high",
  "confidence": 0.92,
  "timestamp": "2025-10-29 14:28:50.123456"
}
```

### 核心字段明细

| 字段           | 类型       | 来源                               | 说明                                                      |
| -------------- | ---------- | ---------------------------------- | --------------------------------------------------------- |
| `camera_id`  | `string` | 任务配置                           | 摄像头唯一标识                                            |
| `message`    | `string` | 动态生成                           | 固定格式 `[{model_name}] 检测到: {class_name}`          |
| `level`      | `string` | `models.json` 的 `alert_level` | 告警级别，见下方枚举                                      |
| `confidence` | `float`  | YOLO 推理结果                      | 最高置信度目标，保留 2 位小数，范围 `0.0 ~ 1.0`         |
| `timestamp`  | `string` | 北京时间 naive                     | 格式 `YYYY-MM-DD HH:MM:SS.ffffff`，**无时区后缀** |

**注意：** timestamp 使用 `datetime.now(UTC+8).replace(tzinfo=None)` 生成，后端入库时也统一转为 naive datetime（北京时区）。

## 三、告警级别枚举

级别配置在 [EdgeAgent/models.json](vscode-webview://0340r3m6cc1a0a60h0keg3qtopjmh9o9vmd5f2bqkujkdlqovg7d/EdgeAgent/models.json) 中定义：

| 模型                     | `alert_level` | 业务含义 |
| ------------------------ | --------------- | -------- |
| `no_mask`（无口罩）    | `"high"`      | 高风险   |
| `no_hat`（无帽子）     | `"medium"`    | 中风险   |
| `no_uniform`（工作服） | `"low"`       | 低风险   |

**Schema 设计注意：** 数据库层面 `level` 是 `String(20)`， **无硬编码枚举约束** 。后端筛选接口使用精确匹配 `==`。建议 schema 定义为枚举或带校验的 string，但不要依赖数据库约束。

## 四、DB_Backend 数据库模型 (SQLAlchemy)

```python
class Alarm(BaseModel):
    __tablename__ = 'alarm'
    id            = Column(String(64),  primary_key=True)       # 格式: ALT{timestamp}{random3}
    camera_id     = Column(String(64),  ForeignKey('camera.id'), nullable=False)
    message       = Column(Text,                              nullable=False)
    level         = Column(String(20),                         nullable=False)
    confidence    = Column(Float,                              nullable=True)
    is_read       = Column(Boolean,     default=False,         nullable=False)
    image_url     = Column(String(255),                        nullable=True)
    detection_tags= Column(Text,                              nullable=True)
    timestamp     = Column(DateTime,                           nullable=False)
    created_at    = Column(DateTime,                           nullable=False)
```

**ID 生成规则：** `ALT{int(time.time())}{uuid[:3].upper()}`，如 `ALT1740500123A7F`

**截图存储：**

* 文件保存于 `/app/alert_images/{uuid}.jpg`
* `image_url` 字段值为 `/alert_images/{uuid}.jpg`（相对路径，前端通过 `/alert_images/...` 直接访问）

## 五、Pydantic Schema（后端校验/序列化）

```python
class AlertBase(BaseModel):
    camera_id:      str
    message:        str
    level:          str
    confidence:     Optional[float] = Field(None, ge=0.0, le=1.0)
    image_url:      Optional[str] = None
    timestamp:      Optional[datetime] = Field(default_factory=beijing_now)
    detection_tags: Optional[str] = None

class AlertResponse(AlertBase):
    id:             str
    created_at:     datetime
    is_read:        bool
    camera_location: Optional[str] = None   # 动态附加，非数据库字段

class AlertPushDTO(AlertResponse):
    """WebSocket 推送用 DTO"""
    camera_name:    str
    location:       str
```

## 六、WebSocket 实时推送格式

后端在告警入库后广播给所有前端连接：

```json
{
  "entity": "alert",
  "action": "create",
  "payload": {
    "id": "ALT1740500123A7F",
    "camera_id": "camera_001",
    "camera_name": "车间A-摄像头1",
    "location": "车间A",
    "camera_location": "车间A",
    "message": "[no_mask] 检测到: no_mask",
    "level": "high",
    "confidence": 0.92,
    "image_url": "/alert_images/uuid.jpg",
    "timestamp": "2025-10-29T14:28:50.123456",
    "created_at": "2025-10-29T14:28:50.123456",
    "detection_tags": null,
    "is_read": false
  }
}
```

**WebSocket 端点：** `ws://host/ws/alerts`（Vite 代理 `/ws` → 后端 8000 端口）

## 七、MQTT 推送格式（第三方平台）

如果配置了 MQTT，告警会额外推送到指定 topic：

```json
{
  "operator": "RecordPush",
  "info": {
    "RecordID": "ALT1740500123A7F",
    "aibox_id": "edge-001",
    "aibox_name": "edge-001",
    "alarm_type": 8,
    "alarm_txt": "[no_mask] 检测到: no_mask",
    "time": "2025-10-29 14:28:50",
    "alarm_chn": 1,
    "pic": "base64编码图片或空字符串"
  }
}
```

* **Topic 默认：** `mqtt/AI-BOX/alarm`
* **图片编码：** Base64 JPEG（当 `use_base64_image=true` 时）
* **代码位置：** [DB_Backend/app/services/mqtt_service.py:261-273](vscode-webview://0340r3m6cc1a0a60h0keg3qtopjmh9o9vmd5f2bqkujkdlqovg7d/DB_Backend/app/services/mqtt_service.py#L261-L273)

## 九、Schema 设计建议

如果你要设计一个新的 schema（如用于数据看板、统计、导出等），核心字段建议：

```typescript
interface Alert {
  id: string;                    // "ALT..." 格式
  cameraId: string;
  cameraName?: string;
  location?: string;
  message: string;
  level: "high" | "medium" | "low" | string;
  confidence: number;            // 0.0 ~ 1.0
  imageUrl?: string;             // "/alert_images/xxx.jpg"
  isRead: boolean;
  detectionTags?: string;        // 预留，目前为空
  timestamp: string;             // 北京时间，格式 "YYYY-MM-DD HH:MM:SS.ffffff"
  createdAt: string;
}
```

如需进一步查看检测对象的完整结构（`detected_objects` 数组中的 bbox、class_name 等），或 AI 配置中的 `alert_threshold` / `ignore_time` / `check_interval` 等参数如何影响告警触发逻辑，可以继续问我。
